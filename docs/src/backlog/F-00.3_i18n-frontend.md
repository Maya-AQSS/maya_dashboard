**Descripción:**
Como usuario del dashboard,
Quiero poder ver la interfaz en mi idioma preferido (español, inglés o valenciano),
Para utilizar el sistema de forma cómoda en el idioma de mi elección.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Idioma por defecto):**
  **Dado que** un usuario accede al dashboard sin preferencia de idioma guardada,
  **Cuando** carga la aplicación,
  **Entonces** la interfaz se muestra en español (ES) por defecto.

- [ ] **Escenario 2 (Cambio de idioma):**
  **Dado que** el usuario está logado y visualiza el dashboard,
  **Cuando** selecciona un idioma diferente (EN o valenciano),
  **Entonces** todos los textos de la interfaz cambian al idioma seleccionado sin recargar la página.

- [ ] **Escenario 3 (Persistencia de preferencia):**
  **Dado que** el usuario ha seleccionado un idioma,
  **Cuando** cierra sesión y vuelve a iniciarla,
  **Entonces** el idioma seleccionado previamente se mantiene.

- [ ] **Escenario 4 (Extensibilidad):**
  **Dado que** la librería i18n está configurada,
  **Cuando** se quiere añadir un nuevo idioma al sistema,
  **Entonces** basta con añadir un nuevo archivo de traducciones sin modificar los componentes.

- [ ] **Requisito Técnico (Cobertura mínima):**
  Todos los textos visibles en la UI deben estar externalizados en archivos de traducciones. No debe quedar ningún texto hardcoded en los componentes React.

---

**MoSCoW:** `Should`
**Sprint:** Sprint 4

**Notas:**

- **Instalación requerida:** Ejecutar `npm install react-i18next i18next` en el directorio `frontend/`.
- Idiomas iniciales: ES (por defecto), EN, valenciano. La arquitectura debe ser extensible a futuros idiomas sin refactoring.
- La preferencia de idioma puede guardarse en localStorage o en el perfil del usuario en backend (decisión de implementación).
- Usar una librería estándar de i18n para React (ej. `react-i18next` o similar).
- Los archivos de traducción se organizan por idioma: `locales/es.json`, `locales/en.json`, `locales/val.json`.
