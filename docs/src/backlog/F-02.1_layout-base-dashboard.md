**Descripción:**
Como usuario autenticado,
Quiero ver una página de inicio con una estructura clara (alerta de fichaje en la parte superior, sidebar izquierdo y área central de widgets),
Para orientarme rápidamente y acceder a las funcionalidades principales del dashboard.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Estructura visual correcta):**
  **Dado que** el usuario está autenticado y accede a la URL raíz del dashboard,
  **Cuando** la página carga,
  **Entonces** se muestra la estructura: zona superior (topbar / alerta de fichaje), sidebar izquierdo colapsable y área central destinada a los widgets.

- [ ] **Escenario 2 (Página de inicio tras login):**
  **Dado que** el usuario completa el flujo de autenticación OIDC,
  **Cuando** Keycloak redirige de vuelta al dashboard,
  **Entonces** el usuario aterriza directamente en el dashboard principal (no en una pantalla de bienvenida intermedia).

- [ ] **Escenario 3 (Tokens de diseño aplicados):**
  **Dado que** el layout usa los tokens de diseño Odoo 19,
  **Cuando** se renderiza el dashboard,
  **Entonces** los colores, tipografía y espaciados son coherentes con el design system Odoo 19 tanto en modo claro como en modo oscuro.

- [ ] **Escenario 4 (Zonas bien definidas):**
  **Dado que** el layout está implementado,
  **Cuando** se integran el sidebar (F-06.1), los widgets (F-02.3) y la alerta de fichaje (F-02.2),
  **Entonces** cada zona ocupa su espacio asignado sin solapamientos ni desbordamientos.

- [ ] **Requisito Técnico (Base para otras features):**
  El layout debe ser el componente contenedor que usan todas las páginas del dashboard. El sidebar y el área de widgets son ranuras (`slots`) del layout, no partes fijas del mismo componente.

---

**MoSCoW:** `Must`
**Sprint:** Sprint 2

**Notas:**

- La alerta de fichaje (F-02.2) debe ser visible en todas las páginas del dashboard, no solo en el inicio — se integra como parte persistente del layout.
- El sidebar (F-06.1) se integra en el layout pero se desarrolla de forma independiente.
- Breakpoints definidos: <768px móvil (1 col), 768-1024px tablet, >1024px escritorio.
- La estructura de 12 columnas para el grid de widgets (F-02.3) se integra en el área central de este layout.
