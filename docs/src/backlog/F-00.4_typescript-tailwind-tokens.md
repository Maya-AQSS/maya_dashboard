**Descripción:**
Como desarrollador frontend,
Quiero disponer de TypeScript estricto, Tailwind 4 configurado y los tokens de diseño Odoo 19 aplicados,
Para desarrollar componentes con tipado seguro y un sistema de diseño coherente con el ecosistema Odoo.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (TypeScript activo):**
  **Dado que** el proyecto React está configurado,
  **Cuando** se compila el frontend,
  **Entonces** TypeScript está en modo estricto y todos los archivos `.tsx` / `.ts` se compilan sin errores de tipo.

- [ ] **Escenario 2 (Tailwind operativo):**
  **Dado que** la aplicación React está corriendo,
  **Cuando** se aplican clases de Tailwind en un componente,
  **Entonces** los estilos se aplican correctamente incluyendo las clases de los tokens Odoo 19 (colores, tipografía).

- [ ] **Escenario 3 (Dark mode):**
  **Dado que** el sistema de dark mode de Tailwind está implementado (ya existente en rama `develop`),
  **Cuando** el usuario activa el modo oscuro,
  **Entonces** todos los componentes de la interfaz respetan el tema oscuro usando las clases `dark:` de Tailwind.

- [ ] **Escenario 4 (Tokens disponibles):**
  **Dado que** los tokens de color y tipografía de Odoo 19 están aplicados,
  **Cuando** un desarrollador quiere usar un color de la paleta Odoo,
  **Entonces** dispone de variables CSS / clases Tailwind personalizadas que referencian los tokens del design system Odoo 19.

- [ ] **Requisito Técnico (Consistencia):**
  La configuración TypeScript y Tailwind se mantiene en la rama base. Si se necesitan tokens adicionales no presentes, deben extraerse del codebase React existente en la rama `develop`.

**Notas:**

- **Migración/Instalación requerida:** Al reestructurar el frontend, asegurarse de mantener dependencias (o instalar de cero): `npm install tailwindcss @tailwindcss/vite` e integrarlo en `vite.config.ts`.
- Los tokens de color y tipografía de Odoo 19 ya están aplicados en el dashboard React actual (rama `develop`). Este backlog valida y documenta esa configuración, no la crea desde cero. Conservar el archivo global CSS (`index.css` o equivalente) donde se importan `@theme` tokens de Tailwind v4.
- Dark mode de Tailwind ya implementado en `develop` — mantener el sistema existente.
- TypeScript es obligatorio en todo el frontend (`tsconfig.app.json`, `tsconfig.node.json`); no se permite JavaScript puro.
- Vite 7 es el bundler. La configuración de TypeScript debe ser compatible con las opciones de Vite.
- Si se detectan tokens adicionales necesarios durante el desarrollo, extraerlos del codebase existente antes de inventarlos.
