**Descripción:**
Como usuario del dashboard,
Quiero recibir notificaciones urgentes en tiempo real mediante un toast sin importar en qué página me encuentre,
Para enterarme inmediatamente de avisos críticos sin tener que revisar activamente la bandeja de notificaciones.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Toast en notificación urgente):**
  **Dado que** el usuario está en cualquier página del dashboard con sesión activa,
  **Cuando** el backend recibe una notificación con `priority: urgente` para ese usuario,
  **Entonces** el frontend muestra un toast visible con el título y cuerpo de la notificación en menos de 2 segundos desde su inserción.

- [ ] **Escenario 2 (Canal privado por usuario):**
  **Dado que** el sistema usa canales WebSocket privados,
  **Cuando** el frontend se suscribe al canal del usuario,
  **Entonces** cada usuario solo recibe las notificaciones urgentes dirigidas a él, no las de otros usuarios.

- [ ] **Escenario 3 (Reconexión automática):**
  **Dado que** la conexión WebSocket se interrumpe temporalmente,
  **Cuando** la conexión se recupera,
  **Entonces** el cliente se reconecta automáticamente y vuelve a suscribirse al canal del usuario.

- [ ] **Escenario 4 (Toast solo para urgentes):**
  **Dado que** se inserta una notificación con `priority: normal` o `priority: aviso`,
  **Cuando** el backend la procesa,
  **Entonces** NO se emite evento WebSocket — el usuario la verá la próxima vez que abra la bandeja de notificaciones.

- [ ] **Escenario 5 (Visibilidad global del toast):**
  **Dado que** el toast está configurado como overlay global,
  **Cuando** aparece una notificación urgente,
  **Entonces** el toast es visible independientemente de la ruta o página en que se encuentre el usuario.

- [ ] **Requisito Técnico (Capacidad):**
  Con 200 usuarios concurrentes, Reverb debe gestionar todas las conexiones WebSocket sin degradación de rendimiento. Una única instancia de Reverb es suficiente para este volumen.

**Notas:**

- **Instalación de Laravel Reverb:** `php artisan install:broadcasting` en el backend para habilitar los WebSockets.
- **Instalación en Frontend:** Instalar Laravel Echo y Pusher js mediante `npm install --save laravel-echo pusher-js`.
- Tecnología: Laravel Reverb (nativo Laravel 11+, incluido en Laravel 13).
- El frontend usa la librería de Echo o similar para suscribirse a los canales privados de Reverb.
- Sin colas internas de Laravel — la emisión WebSocket ocurre directamente al procesar el endpoint de inserción.
- El toast desaparece automáticamente tras unos segundos (tiempo estándar UX — decisión de implementación frontend).
