**Descripción:**
Como usuario del dashboard,
Quiero registrar mi entrada y salida laboral con una confirmación visual clara y saber qué hacer si el sistema está caído,
Para tener constancia de mis fichajes y no quedarme sin alternativa si Odoo no está disponible.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Fichaje de entrada exitoso):**
  **Dado que** el usuario no ha fichado entrada en el día actual,
  **Cuando** hace clic en "Fichar entrada",
  **Entonces** se envía la petición al adapter de Odoo, el fichaje se registra y aparece un mensaje de confirmación de éxito que desaparece automáticamente entre 3 y 5 segundos.

- [ ] **Escenario 2 (Fichaje de salida exitoso):**
  **Dado que** el usuario ya fichó entrada y quiere registrar su salida,
  **Cuando** hace clic en "Fichar salida",
  **Entonces** se registra el fichaje de salida en Odoo y aparece confirmación visual con auto-cierre entre 3 y 5 segundos.

- [ ] **Escenario 3 (Odoo caído — modal de aviso):**
  **Dado que** el usuario intenta fichar y el servicio Odoo no está disponible,
  **Cuando** el adapter devuelve un error de conexión,
  **Entonces** se muestra un modal de cierre manual (no se cierra automáticamente) informando que el usuario debe fichar presencialmente en la entrada al centro.

- [ ] **Escenario 4 (Auditoría de fichaje):**
  **Dado que** el usuario realiza un fichaje (entrada, salida o error),
  **Cuando** la acción se completa o falla,
  **Entonces** se registra el evento correspondiente en el audit log (F-08.1) con el tipo `FICHAJE`.

- [ ] **Requisito Técnico (Desacoplamiento de la UI):**
  Los componentes de UI de fichaje no llaman directamente a ninguna API externa. Toda la comunicación con Odoo pasa exclusivamente por el adapter de F-04.1.

**Notas:**

- El modal de "Odoo caído" es de cierre manual (el usuario debe cerrarlo explícitamente), a diferencia del mensaje de éxito que se cierra automáticamente.
- Los tipos de fichaje en esta fase: entrada y salida. Ampliable en futuras versiones.
- La alerta de fichaje pendiente del banner (F-02.2) se actualiza tras un fichaje de entrada exitoso.
