**Descripción:**
Como usuario del dashboard con cuenta Microsoft corporativa,
Quiero vincular mi cuenta Microsoft a través de Keycloak para que el sistema pueda acceder a mi calendario de Outlook,
Para ver mis eventos en el widget de calendario del dashboard sin gestionar credenciales adicionales.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Flujo Token Exchange):**
  **Dado que** el usuario tiene una sesión activa en Keycloak,
  **Cuando** quiere vincular su cuenta Microsoft,
  **Entonces** Keycloak actúa como Identity Broker y ejecuta el Token Exchange RFC 8693 para obtener un token de acceso a Microsoft Graph en nombre del usuario.

- [ ] **Escenario 2 (Cuenta vinculada — acceso a Calendar):**
  **Dado que** el usuario ha vinculado su cuenta Microsoft,
  **Cuando** el `MicrosoftGraphCalendarProvider` realiza llamadas a la API de Graph,
  **Entonces** usa el token obtenido mediante Token Exchange para autenticarse en Microsoft Graph con éxito.

- [ ] **Escenario 3 (Token expirado — renovación):**
  **Dado que** el token de Microsoft puede expirar,
  **Cuando** el sistema detecta que el token de acceso a Graph ha expirado,
  **Entonces** realiza el Token Exchange para obtener un token renovado sin requerir intervención del usuario.

- [ ] **Escenario 4 (Sin cuenta vinculada — estado informativo):**
  **Dado que** el usuario no ha vinculado su cuenta Microsoft,
  **Cuando** el widget de calendario intenta obtener eventos,
  **Entonces** muestra el estado "conecta tu cuenta Microsoft" con instrucciones para iniciar el proceso de vinculación.

- [ ] **Requisito Técnico (Keycloak como broker):**
  La autenticación con Microsoft pasa exclusivamente por Keycloak Identity Brokering. El dashboard no gestiona directamente credenciales OAuth2 de Microsoft Azure AD.

**Notas:**

- Opción de autenticación: Identity Brokering via Keycloak (Opción A). Token Exchange RFC 8693 para Microsoft Graph.
- Esta feature depende de que el realm Keycloak esté configurado con el Identity Provider de Microsoft. Coordinar con infra (`IaC.git`).
- Solo lectura: el token obtenido solo se usa para leer eventos del calendario, no para escribir.
- El widget de calendario (F-05.3) y esta feature son ambos `Could` — se implementan conjuntamente cuando la integración esté operativa.
