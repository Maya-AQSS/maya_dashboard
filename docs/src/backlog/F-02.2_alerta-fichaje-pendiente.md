**Descripción:**
Como usuario del dashboard,
Quiero ver un aviso destacado cuando aún no he fichado la entrada del día,
Para no olvidarme de registrar mi presencia y no incurrir en incidencias de control horario.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Alerta visible cuando no ha fichado):**
  **Dado que** el usuario está autenticado y no ha realizado el fichaje de entrada del día actual,
  **Cuando** accede a cualquier página del dashboard,
  **Entonces** se muestra un banner de alerta en la zona superior de la pantalla indicando que aún no ha fichado.

- [ ] **Escenario 2 (Alerta oculta cuando ha fichado):**
  **Dado que** el usuario ya ha realizado el fichaje de entrada del día,
  **Cuando** carga cualquier página del dashboard,
  **Entonces** el banner de alerta no aparece.

- [ ] **Escenario 3 (Visibilidad en todas las páginas):**
  **Dado que** el banner está integrado en el layout base,
  **Cuando** el usuario navega entre distintas secciones del dashboard (notificaciones, perfil, herramientas, etc.),
  **Entonces** el banner de alerta permanece visible en todas las páginas mientras no haya fichado.

- [ ] **Escenario 4 (Actualización dinámica):**
  **Dado que** el usuario realizó el fichaje de entrada desde el widget de fichajes,
  **Cuando** el fichaje se confirma correctamente,
  **Entonces** el banner de alerta desaparece sin necesidad de recargar la página.

- [ ] **Requisito Técnico (Consulta al adapter Odoo):**
  La lógica de visibilidad del banner debe usar la capa de integración con Odoo (F-04.1), no consultar directamente ninguna API. El adapter gestiona si usar el mock o la API real.

---

**MoSCoW:** `Must`

**Notas:**

- La alerta solo aparece cuando el usuario NO ha fichado entrada ese día. Si ya fichó entrada (aunque no haya fichado salida), el banner no se muestra.
- El banner es parte del layout (F-02.1) y no desaparece al navegar entre páginas.
- La consulta del estado de fichaje se realiza al cargar el layout, no solo el dashboard principal.
