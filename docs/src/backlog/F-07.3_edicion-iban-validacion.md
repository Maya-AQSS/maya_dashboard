**Descripción:**
Como usuario del dashboard,
Quiero poder añadir o actualizar mi IBAN en mi perfil con validación de formato,
Para tener registrado mi número de cuenta bancaria europeo de forma correcta para trámites del CEED.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Edición directa del IBAN):**
  **Dado que** el usuario accede a la sección de IBAN en su perfil,
  **Cuando** introduce o modifica su IBAN y guarda,
  **Entonces** el cambio se persiste directamente en `registros_ceed` sin flujo de aprobación.

- [ ] **Escenario 2 (Validación de formato IBAN europeo):**
  **Dado que** el usuario introduce un IBAN,
  **Cuando** envía el formulario,
  **Entonces** el sistema valida que el valor cumple el formato IBAN europeo (longitud correcta según país, checksum IBAN válido) antes de guardarlo.

- [ ] **Escenario 3 (Error de validación — IBAN inválido):**
  **Dado que** el usuario introduce un IBAN con formato incorrecto (ej. checksum fallido o longitud incorrecta),
  **Cuando** intenta guardar,
  **Entonces** el sistema muestra un mensaje de error indicando que el IBAN no es válido y no persiste el valor.

- [ ] **Escenario 4 (Auditoría del cambio):**
  **Dado que** el IBAN es un dato financiero sensible,
  **Cuando** el usuario guarda un IBAN nuevo o modificado,
  **Entonces** la acción queda registrada en el audit log (F-08.1) con el tipo `PROFILE` indicando que el campo IBAN fue modificado.

- [ ] **Escenario 5 (IBAN vacío permitido):**
  **Dado que** el usuario puede no tener IBAN registrado,
  **Cuando** el campo IBAN está vacío en el perfil,
  **Entonces** la pantalla lo muestra como no informado, sin error de validación.

- [ ] **Requisito Técnico (Validación en backend):**
  La validación del formato IBAN debe realizarse en el backend (Laravel), no solo en el frontend. El frontend puede mostrar validación previa, pero la validación definitiva ocurre en el servidor.

---

**MoSCoW:** `Should`
**Sprint:** Sprint 4

**Notas:**

- El IBAN es editable directamente por el usuario. Sin flujo de aprobación.
- Validación: cualquier IBAN europeo válido (no solo español). El algoritmo de validación IBAN incluye: código de país (2 letras), dígitos de control (2 dígitos) y BBAN según longitud del país.
- No se debe mostrar el IBAN completo en la interfaz por razones de seguridad — considerar enmascaramiento parcial (ej. `ES** **** **** **** **** 1234`).
