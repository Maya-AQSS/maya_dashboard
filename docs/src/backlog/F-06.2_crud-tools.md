**Descripción:**
Como usuario del dashboard,
Quiero poder crear, editar y eliminar Tools en el catálogo de aplicaciones,
Para mantener actualizado el listado de herramientas disponibles para todos los usuarios.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Crear Tool):**
  **Dado que** el usuario accede al formulario de creación de Tool,
  **Cuando** rellena los campos obligatorios (nombre, URL, tipo) y guarda,
  **Entonces** la Tool se crea en el catálogo y aparece disponible para todos los usuarios.

- [ ] **Escenario 2 (Editar Tool):**
  **Dado que** existe una Tool en el catálogo,
  **Cuando** el usuario edita sus campos y guarda los cambios,
  **Entonces** la Tool se actualiza y los cambios son visibles de inmediato para todos los usuarios.

- [ ] **Escenario 3 (Eliminar Tool):**
  **Dado que** existe una Tool en el catálogo,
  **Cuando** el usuario la elimina y confirma la acción,
  **Entonces** la Tool desaparece del catálogo y de los favoritos de cualquier usuario que la tuviese marcada.

- [ ] **Escenario 4 (Campos de la Tool):**
  **Dado que** se crea o edita una Tool,
  **Cuando** se guardan los datos,
  **Entonces** el registro incluye: `nombre` (texto), `icono` (URL o identificador de icono), `url` (enlace de acceso), `tipo` (interno/externo).

- [ ] **Escenario 5 (Acceso de todos los usuarios al CRUD):**
  **Dado que** los roles y permisos no están implementados en esta fase,
  **Cuando** cualquier usuario autenticado accede al CRUD,
  **Entonces** puede crear, editar y eliminar Tools sin restricciones.

- [ ] **Requisito Técnico (Sin categorías ni límite de catálogo):**
  Las Tools no tienen categorías en esta fase. No hay límite máximo de Tools en el catálogo.

---

**MoSCoW:** `Must`
**Sprint:** Sprint 2

**Notas:**

- El catálogo se gestionará exclusivamente desde el panel CRUD — no hay catálogo inicial predefinido.
- Todos los usuarios pueden crear/editar/eliminar Tools mientras no existan roles y permisos (v2).
- Sin categorías de Tools en esta fase.
- Sin límite máximo de Tools en el catálogo.
- El tipo `interno` implica navegación dentro del dashboard (React Router). El tipo `externo` implica apertura en nueva pestaña.
- Al eliminar una Tool, se deben eliminar también todas las referencias de favoritos de cualquier usuario.
