**Descripción:**
Como usuario del dashboard,
Quiero acceder a una página con el catálogo completo de Tools donde pueda buscar, gestionar mis favoritos y administrar el catálogo,
Para tener un punto central de gestión de todas las herramientas disponibles.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Listado completo de Tools):**
  **Dado que** el usuario accede a la página de herramientas,
  **Cuando** la página carga,
  **Entonces** se muestra el catálogo completo de Tools disponibles con sus datos (nombre, icono, tipo) y el indicador de si es favorita del usuario.

- [ ] **Escenario 2 (Búsqueda de Tools):**
  **Dado que** el catálogo puede tener muchas Tools,
  **Cuando** el usuario escribe en el campo de búsqueda,
  **Entonces** el listado se filtra en tiempo real mostrando solo las Tools cuyo nombre contiene el texto buscado.

- [ ] **Escenario 3 (Acceso al CRUD de gestión):**
  **Dado que** el usuario quiere crear o editar una Tool,
  **Cuando** hace clic en "Nueva Tool" o en el botón de edición de una Tool existente,
  **Entonces** accede al formulario de creación/edición de la Tool (F-06.2).

- [ ] **Escenario 4 (Gestión de favoritos desde la página):**
  **Dado que** el usuario quiere cambiar sus favoritos,
  **Cuando** interactúa con el indicador de favorito de una Tool en el listado,
  **Entonces** puede marcar o desmarcar la Tool como favorita sin salir de la página (F-06.3).

- [ ] **Escenario 5 (Reordenar favoritas con drag & drop):**
  **Dado que** el usuario tiene Tools marcadas como favoritas,
  **Cuando** arrastra una favorita para cambiar su posición,
  **Entonces** el nuevo orden se guarda y se refleja en el sidebar inmediatamente.

- [ ] **Requisito Técnico (Auditoría de acceso a Tools):**
  Cada acceso a una Tool (navegación al hacer clic) debe registrarse en el audit log (F-08.1) con el tipo `TOOLS`.

---

**MoSCoW:** `Must`
**Sprint:** Sprint 3

**Notas:**

- La gestión de favoritos (marcar/desmarcar y reordenar) se realiza directamente desde esta página, no desde un modal.
- La búsqueda puede implementarse en frontend (filtrado local) si el catálogo es razonablemente pequeño, o con llamada al backend si crece.
- El acceso al CRUD de gestión debe estar disponible para todos los usuarios (sin roles en esta fase).
