**Descripción:**
Como administrador del sistema,
Quiero que todas las acciones críticas de usuario queden registradas automáticamente en la base de datos con retención de 1 año,
Para disponer de un historial auditable de la actividad del sistema.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Registro de acciones de autenticación):**
  **Dado que** un usuario realiza una acción de autenticación,
  **Cuando** completa el login correctamente, falla el login o hace logout,
  **Entonces** se registra un evento `AUTH` en el audit log con: usuario, tipo de evento (login_ok/login_failed/logout), timestamp y metadatos relevantes.

- [ ] **Escenario 2 (Registro de acciones de perfil):**
  **Dado que** un usuario edita un campo sensible de su perfil (IBAN, teléfono, dirección, matrícula),
  **Cuando** guarda el cambio,
  **Entonces** se registra un evento `PROFILE` indicando qué campo fue modificado (sin incluir el valor anterior ni nuevo por seguridad).

- [ ] **Escenario 3 (Registro de acciones de fichaje):**
  **Dado que** un usuario registra un fichaje (entrada, salida o error),
  **Cuando** la acción se completa o falla,
  **Entonces** se registra un evento `FICHAJE` con el tipo de acción y resultado.

- [ ] **Escenario 4 (Registro de notificaciones, Tools y Calendar):**
  **Dado que** el usuario marca notificaciones como leídas, accede a una Tool o conecta su cuenta de calendario,
  **Cuando** realiza esas acciones,
  **Entonces** se registra el evento correspondiente: `NOTIFICACIONES` (leída/todas leídas), `TOOLS` (acceso/favorita), `CALENDAR` (conectar cuenta).

- [ ] **Escenario 5 (Retención y limpieza automática):**
  **Dado que** la política de retención es de 1 año,
  **Cuando** un registro supera 1 año de antigüedad,
  **Entonces** un proceso automático lo elimina de la base de datos (ej. comando programado con scheduler de Laravel).

- [ ] **Requisito Técnico (Librería estándar):**
  Usar `spatie/laravel-activitylog` como librería de auditoría. Los registros se almacenan en la BD local de Laravel (PostgreSQL). Los errores de sistema se envían a RabbitMQ (F-08.3), no a esta tabla.

**Notas:**

- **Instalación requerida:** Instalar la librería mediante `composer require spatie/laravel-activitylog`. Posteriormente, publicar sus migraciones (`php artisan vendor:publish --provider="Spatie\Activitylog\ActivitylogServiceProvider" --tag="activitylog-migrations"`) y ejecutar `php artisan migrate`.
- Acciones auditadas (listado validado): AUTH (login OK/fallido/logout), PROFILE (campo sensible), FICHAJE (entrada/salida/error), NOTIFICACIONES (leída/todas leídas), TOOLS (acceso/favorita), CALENDAR (conectar cuenta).
- Retención: 1 año. Limpieza automática mediante scheduler de Laravel (`php artisan activitylog:clean`).
- Los errores de sistema van a RabbitMQ (F-08.3) — son distintos del audit log de acciones de usuario.
- Acceso MVP: cada usuario solo puede ver su propio historial. Vista global para administradores queda para v2.
