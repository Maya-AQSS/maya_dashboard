**Descripción:**
Como usuario del dashboard,
Quiero que la interfaz se adapte correctamente a cualquier tamaño de pantalla y disponer de un modo compacto para ver más información en pantallas de escritorio,
Para tener una experiencia óptima tanto desde el ordenador como desde el móvil o tablet.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Móvil — 1 columna):**
  **Dado que** el usuario accede al dashboard desde un dispositivo con pantalla inferior a 768px,
  **Cuando** la página carga,
  **Entonces** el layout muestra 1 columna con los widgets apilados verticalmente, el sidebar se oculta (accesible como drawer) y todos los elementos son legibles sin scroll horizontal.

- [ ] **Escenario 2 (Tablet — layout intermedio):**
  **Dado que** el usuario accede desde un dispositivo con pantalla entre 768px y 1024px,
  **Cuando** la página carga,
  **Entonces** el layout se adapta al espacio intermedio usando el grid de forma que los widgets sean legibles y funcionales.

- [ ] **Escenario 3 (Escritorio — grid completo):**
  **Dado que** el usuario accede desde pantalla superior a 1024px,
  **Cuando** la página carga,
  **Entonces** se muestra el grid completo de 12 columnas con el sidebar en modo rail (iconos) y el área de widgets con su layout completo.

- [ ] **Escenario 4 (Vista compacta activable):**
  **Dado que** el usuario está en escritorio,
  **Cuando** activa la vista compacta,
  **Entonces** los widgets se muestran más pequeños (menor altura), con menos ítems visibles por widget, permitiendo ver más información en la misma pantalla.

- [ ] **Requisito Técnico (Breakpoints con Tailwind):**
  Los breakpoints deben usar las clases de Tailwind: `sm:` (≥640px), `md:` (≥768px), `lg:` (≥1024px). Los tamaños exactos de referencia son: móvil <768px, tablet 768-1024px, escritorio >1024px.

**Notas:**

- Vista compacta solo disponible en escritorio. En móvil/tablet no aplica.
- La vista compacta reduce el tamaño visual de los widgets pero no elimina funcionalidad.
- El sidebar en móvil se convierte en drawer/overlay (gestionado por F-06.1).
