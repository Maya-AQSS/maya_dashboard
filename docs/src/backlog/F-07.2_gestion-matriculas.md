**Descripción:**
Como usuario del dashboard,
Quiero gestionar mis matrículas de vehículo (añadir, editar y eliminar),
Para tener registradas las matrículas de mis vehículos en mi perfil del CEED.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Listado de matrículas):**
  **Dado que** el usuario accede a la sección de matrículas de su perfil,
  **Cuando** la sección carga,
  **Entonces** se muestran todas las matrículas registradas por ese usuario.

- [ ] **Escenario 2 (Añadir matrícula):**
  **Dado que** el usuario quiere registrar un vehículo,
  **Cuando** introduce el valor de la matrícula y guarda,
  **Entonces** la matrícula se añade a la tabla de matrículas del usuario y aparece en el listado.

- [ ] **Escenario 3 (Eliminar matrícula):**
  **Dado que** el usuario tiene matrículas registradas,
  **Cuando** elimina una matrícula y confirma la acción,
  **Entonces** la matrícula se borra de la base de datos y desaparece del listado.

- [ ] **Escenario 4 (Múltiples matrículas por usuario):**
  **Dado que** un usuario puede tener varios vehículos,
  **Cuando** añade más de una matrícula,
  **Entonces** el sistema las almacena correctamente en la tabla separada de matrículas (relación uno a muchos con el usuario).

- [ ] **Escenario 5 (Auditoría de cambios):**
  **Dado que** las matrículas son datos sensibles,
  **Cuando** se añade o elimina una matrícula,
  **Entonces** la acción queda registrada en el audit log (F-08.1) con el tipo `PROFILE`.

- [ ] **Requisito Técnico (Sin validación de formato):**
  El sistema no valida el formato de la matrícula (española, europea, motos, etc.). Se almacena cualquier texto introducido por el usuario.

---

**MoSCoW:** `Must`
**Sprint:** Sprint 3

**Notas:**

- Las matrículas se almacenan en una tabla separada (no en `registros_ceed`), con relación de clave foránea al usuario.
- Sin validación de formato de matrícula. Se acepta cualquier texto.
- Un usuario puede tener múltiples matrículas. Sin límite máximo.
- La edición de una matrícula puede implementarse como eliminar + añadir, o con un campo editable directo — decisión de UX del equipo.
