**Descripción:**
Como usuario del dashboard,
Quiero ver un resumen de mis últimos fichajes en el widget del dashboard,
Para comprobar rápidamente mi historial de entradas y salidas sin necesidad de acceder a Odoo directamente.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Historial reciente visible):**
  **Dado que** el usuario accede al dashboard,
  **Cuando** el widget de fichajes carga,
  **Entonces** muestra el historial reciente de fichajes del usuario (últimas entradas/salidas) consultado directamente desde Odoo mediante el adapter.

- [ ] **Escenario 2 (Estado del día actual):**
  **Dado que** el widget muestra el estado de fichaje,
  **Cuando** el usuario ya ha fichado entrada ese día,
  **Entonces** el widget indica claramente el estado del día: hora de entrada registrada y, si aplica, hora de salida.

- [ ] **Escenario 3 (Odoo no disponible):**
  **Dado que** el servicio Odoo no está accesible al cargar el widget,
  **Cuando** el adapter devuelve un error,
  **Entonces** el widget muestra un mensaje informativo de que los datos no están disponibles temporalmente, sin bloquear el resto del dashboard.

- [ ] **Escenario 4 (Sin historial previo):**
  **Dado que** el usuario no tiene fichajes registrados,
  **Cuando** el widget carga,
  **Entonces** muestra un estado vacío informativo ("Aún no tienes fichajes registrados" o similar).

- [ ] **Requisito Técnico (Solo datos de Odoo):**
  El historial siempre se consulta desde Odoo (mock o real). No existe copia local en la BD del dashboard. El widget no puede mostrar datos que no provengan del adapter.

**Notas:**

- El widget es uno de los 3 fijos del MVP. Su posición en el grid se define en F-02.3.
- El número de fichajes recientes mostrados (ej. últimos 5 o últimos 7 días) es decisión de UX del equipo.
- Las acciones de fichaje (botones de entrada/salida) se implementan en F-04.2 y pueden integrarse también en este widget o estar en una sección separada — decisión de diseño.
