**Descripción:**
Como usuario del dashboard,
Quiero poder ver mis notificaciones, marcarlas como leídas y acceder a las acciones asociadas,
Para gestionar mis avisos y navegar a las secciones relevantes desde las notificaciones.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Listado de notificaciones):**
  **Dado que** el usuario accede a la sección de notificaciones,
  **Cuando** la lista carga,
  **Entonces** se muestran sus notificaciones ordenadas de más reciente a más antigua, distinguiendo visualmente entre leídas y no leídas.

- [ ] **Escenario 2 (Marcar como leída):**
  **Dado que** el usuario tiene notificaciones no leídas,
  **Cuando** hace clic sobre una notificación individual,
  **Entonces** la notificación se marca como leída (se persiste `read_at` en backend) y el indicador visual de no leída desaparece.

- [ ] **Escenario 3 (Marcar todas como leídas):**
  **Dado que** el usuario tiene múltiples notificaciones no leídas,
  **Cuando** ejecuta la acción "Marcar todas como leídas",
  **Entonces** todas sus notificaciones se marcan como leídas en backend y la interfaz se actualiza.

- [ ] **Escenario 4 (Acción de redirección):**
  **Dado que** una notificación tiene una `action_url` configurada,
  **Cuando** el usuario hace clic en la acción de la notificación,
  **Entonces** es redirigido a la URL correspondiente: si es ruta interna, navega dentro del dashboard; si es URL externa, abre en nueva pestaña.

- [ ] **Escenario 5 (Filtros):**
  **Dado que** el usuario accede a la lista de notificaciones,
  **Cuando** aplica filtros (ej. solo no leídas, por tipo o prioridad),
  **Entonces** la lista se filtra según el criterio seleccionado. Todos los usuarios pueden acceder a los filtros en esta fase.

- [ ] **Requisito Técnico (Auditoría):**
  Las acciones "leída" y "todas leídas" deben registrarse en el audit log (F-08.1) con el evento `NOTIFICACIONES`.

**MoSCoW:** `Must`

**Notas:**

- El estado leído/no leído se persiste en backend. No se gestiona solo en frontend.
- Las acciones redirigen a rutas internas o URLs externas según el campo `action_url` de la notificación.
- Todos los usuarios pueden usar los filtros en MVP. Con roles (v2) se restringirá.
- La paginación del listado es decisión de implementación del equipo.
