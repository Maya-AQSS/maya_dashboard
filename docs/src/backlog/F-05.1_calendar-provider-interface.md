**Descripción:**
Como desarrollador backend,
Quiero definir una interface genérica `CalendarProvider` con su implementación concreta para Microsoft Graph,
Para poder activar la integración del calendario Outlook en el futuro sin necesidad de refactoring.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Interface genérica definida):**
  **Dado que** el sistema necesita ser extensible a múltiples proveedores de calendario,
  **Cuando** se diseña la arquitectura,
  **Entonces** existe una interface `CalendarProvider` que define los métodos: obtener eventos del día, obtener eventos por rango de fechas.

- [ ] **Escenario 2 (Implementación Microsoft Graph creada):**
  **Dado que** el proveedor inicial es Microsoft Graph,
  **Cuando** se implementa la interface,
  **Entonces** existe una clase concreta `MicrosoftGraphCalendarProvider` que implementa `CalendarProvider` usando la API de Microsoft Graph (aunque permanezca inactiva en UI hasta F-05.3).

- [ ] **Escenario 3 (Proveedor nulo para estado desconectado):**
  **Dado que** el widget está omitido de la UI hasta que la integración esté disponible,
  **Cuando** se usa un proveedor nulo o stub,
  **Entonces** el sistema devuelve un estado "no disponible" sin errores ni excepciones.

- [ ] **Escenario 4 (Configuración por binding):**
  **Dado que** la interface está configurada en el Service Container,
  **Cuando** se inyecta `CalendarProvider` en un controlador o servicio,
  **Entonces** el binding puede cambiarse sin modificar el código del consumidor.

- [ ] **Requisito Técnico (Solo lectura):**
  La interface solo define operaciones de lectura — obtener eventos. No incluye métodos para crear ni modificar eventos.

---

**MoSCoW:** `Should`
**Sprint:** Sprint 4

**Notas:**

- El widget de calendario (F-05.3) está omitido en la UI hasta que la integración Microsoft Graph esté operativa. Sin embargo, la arquitectura del provider debe estar lista para activarlo sin refactoring.
- La autenticación con Microsoft (F-05.2) es una feature separada de tipo `Could` — la interface puede diseñarse sin bloquear en esa feature.
- Solo lectura: no permite crear ni modificar eventos de Outlook desde el dashboard.
- Sin cuenta Microsoft vinculada: devolver estado "conecta tu cuenta" (gestionado por F-05.3).
