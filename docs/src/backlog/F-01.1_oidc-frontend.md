**Descripción:**
Como usuario del dashboard,
Quiero autenticarme mediante mi cuenta corporativa a través de Keycloak,
Para acceder de forma segura al dashboard sin gestionar contraseñas adicionales.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Login exitoso):**
  **Dado que** el usuario no está autenticado y accede al dashboard,
  **Cuando** hace clic en "Iniciar sesión",
  **Entonces** es redirigido al servidor Keycloak, completa la autenticación con sus credenciales corporativas y vuelve al dashboard con sesión activa.

- [ ] **Escenario 2 (PKCE activo):**
  **Dado que** el flujo de autenticación usa Authorization Code + PKCE,
  **Cuando** se inicia el flujo,
  **Entonces** se genera un `code_verifier` y `code_challenge` únicos por sesión y el intercambio de código se realiza correctamente sin exponer secretos en el cliente.

- [ ] **Escenario 3 (Protección de rutas):**
  **Dado que** el usuario no está autenticado,
  **Cuando** intenta acceder directamente a cualquier ruta del dashboard,
  **Entonces** es redirigido automáticamente al flujo de login de Keycloak.

- [ ] **Escenario 4 (Logout):**
  **Dado que** el usuario está autenticado,
  **Cuando** hace clic en "Cerrar sesión",
  **Entonces** la sesión local se destruye y se invoca el endpoint de logout de Keycloak para invalidar la sesión en el servidor.

- [ ] **Requisito Técnico (Cliente público):**
  El cliente Keycloak del frontend es de tipo `public` — no requiere client secret. El flujo PKCE es la medida de seguridad para clientes públicos.

**Notas:**

- Cliente Keycloak frontend: tipo `public`, Authorization Code + PKCE, sin client secret.
- Redirect URI local: `http://localhost:5173`. Parametrizar por entorno (`VITE_KEYCLOAK_REDIRECT_URI`).
- El realm Keycloak ya existe en producción. No modificar su estructura sin coordinación con infra.
- La política de contraseñas es responsabilidad de Keycloak; el frontend no la implementa.
- Roles existen en Keycloak pero no se implementan en esta fase — el token los transporta pero el frontend no los consume.
- 2FA no previsto para esta versión.
