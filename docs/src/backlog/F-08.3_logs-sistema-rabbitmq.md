**Descripción:**
Como equipo de operaciones,
Quiero que los errores de sistema del dashboard se publiquen automáticamente a RabbitMQ,
Para centralizar la monitorización de errores y recibir alertas desde el sistema externo de observabilidad.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Error de sistema publicado a RabbitMQ):**
  **Dado que** el sistema genera un error interno (excepción no controlada, fallo de integración, etc.),
  **Cuando** Laravel captura el error,
  **Entonces** publica un mensaje en RabbitMQ con el esquema de log definido: timestamp, nivel de severidad, mensaje, contexto y stack trace (si aplica).

- [ ] **Escenario 2 (URL de RabbitMQ configurable):**
  **Dado que** la URL de RabbitMQ para logs de sistema aún no está disponible,
  **Cuando** se configura el sistema,
  **Entonces** la URL se define exclusivamente mediante variable de entorno (`RABBITMQ_LOGS_URL` o similar) y el código funciona con la URL simulada sin errores.

- [ ] **Escenario 3 (RabbitMQ no disponible — sin bloqueo):**
  **Dado que** el servicio RabbitMQ puede estar temporalmente caído,
  **Cuando** el sistema intenta publicar un error y falla,
  **Entonces** el fallo de publicación no bloquea el flujo principal de la aplicación y se registra localmente el intento fallido.

- [ ] **Escenario 4 (Esquema de log definido libremente):**
  **Dado que** no existe un contrato externo impuesto para el esquema del mensaje,
  **Cuando** se diseña el payload del mensaje de error,
  **Entonces** el esquema incluye al menos: `timestamp`, `level` (error/warning/critical), `message`, `context` (objeto con datos relevantes) y `service` (identificador del dashboard).

- [ ] **Requisito Técnico (Librería php-amqplib):**
  Usar `php-amqplib` para la conexión y publicación en RabbitMQ desde Laravel.

**MoSCoW:** `Must`
**Sprint:** Sprint 2

**Notas:**

- **Instalación requerida:** Ejecutar `composer require php-amqplib/php-amqplib` en el backend para instalar el cliente oficial de RabbitMQ.
- En el entorno de desarrollo local, se utilizará el contenedor de RabbitMQ provisto por el `docker-compose.yml` unificado (F-00.1). Las variables locales en `.env.example` apuntarán a este contenedor.
- Para el entorno de producción, la URL y credenciales definitivas del cluster de RabbitMQ las proporcionará el equipo de infraestructura, configurándose exclusivamente en el archivo `.env` del servidor real.
- El esquema del mensaje se define libremente en este proyecto — no hay contrato externo. Se recomienda implementar un Custom Log Channel en Laravel que formatee y publique el error usando esta librería.
- Distinción importante: los logs de errores de SISTEMA van a RabbitMQ (esta feature). Las acciones de USUARIO van al audit log en BD local (F-08.1). Son canales distintos.
- Sin colas internas de Laravel — publicación directa al exchange de RabbitMQ externo.
