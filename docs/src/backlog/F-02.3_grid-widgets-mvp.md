**Descripción:**
Como usuario del dashboard,
Quiero ver los tres widgets principales (Notificaciones, Fichajes y Calendario) en un layout de grid fijo,
Para acceder de un vistazo a la información más relevante de mi jornada laboral.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Grid de 12 columnas renderizado):**
  **Dado que** el usuario accede al dashboard principal,
  **Cuando** la página carga,
  **Entonces** el área central muestra un grid de 12 columnas con los 3 widgets en posiciones y tamaños fijos definidos para el MVP.

- [ ] **Escenario 2 (Tamaños discretos de widget):**
  **Dado que** el grid está activo,
  **Cuando** se renderizan los widgets,
  **Entonces** cada widget ocupa un tamaño discreto de la cuadrícula: pequeño (3 col), mediano (6 col) o grande (9 col), con altura fija por tipo de widget.

- [ ] **Escenario 3 (Configuración fija en MVP):**
  **Dado que** el MVP no permite personalización,
  **Cuando** cualquier usuario accede al dashboard,
  **Entonces** todos los usuarios ven exactamente el mismo layout de widgets (misma posición y tamaño) sin opción de reordenarlos o redimensionarlos.

- [ ] **Escenario 4 (Slots vacíos para widgets no disponibles):**
  **Dado que** el widget de Calendario Outlook está omitido en la UI hasta que la integración esté disponible,
  **Cuando** el grid se renderiza,
  **Entonces** la posición del widget de Calendario muestra un estado "no disponible" o simplemente el slot queda vacío, sin errores en la UI.

- [ ] **Requisito Técnico (Extensibilidad para v2):**
  La arquitectura del grid debe permitir añadir personalización drag & drop en v2 sin necesidad de reescribirlo. El MVP usa posiciones fijas, pero la estructura de datos que define qué widget va en qué posición debe ser configurable.

**Notas:**

- MVP: 3 widgets fijos (Notificaciones, Fichajes, Calendario Outlook). Sin personalización drag & drop — explícitamente reservado para v2.
- Los widgets son componentes independientes que se integran en este grid: F-03.4 (Notificaciones), F-04.3 (Fichajes), F-05.3 (Calendario — Could, omitido hasta disponibilidad).
- La tabla `user_widget_configurations` (sync entre dispositivos) queda para v2.
- Breakpoints del grid: móvil <768px → 1 columna (widgets apilados), tablet/escritorio → grid completo.
